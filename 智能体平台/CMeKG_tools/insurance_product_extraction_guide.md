# 保险产品提取方案：基于现有医疗RE模型的复用与调整

## 一、训练代码复用分析

### 1. 代码结构评估

现有`medical_re.py`的代码结构具有良好的通用性，主要包含以下核心组件：

- **数据处理**：`load_data`函数加载JSON格式的训练数据
- **模型定义**：`Model4s`（实体识别）和`Model4po`（关系/对象预测）
- **训练流程**：`train`函数实现完整的训练逻辑
- **评估机制**：`evaluate`函数计算F1、精确率和召回率
- **推理接口**：`extract_spoes`和`get_triples`函数实现模型推理

### 2. 可复用性结论

**可以复用现有训练代码**，但需要针对保险产品领域进行以下调整：

| 组件 | 复用情况 | 调整内容 |
|------|----------|----------|
| 模型结构 | 完全复用 | 无需修改模型架构 |
| 训练流程 | 完全复用 | 无需修改训练逻辑 |
| 数据处理 | 部分复用 | 需要修改数据加载路径和格式验证 |
| 配置参数 | 部分复用 | 需要修改路径、关系类型数量等 |
| 预训练模型 | 不可复用 | 需要更换为保险/金融领域预训练模型 |

## 三、CWS、NER、RE的处理方案

### 1. 中文分词（CWS）

**推荐方案**：使用BERT内置的subword分词机制，无需单独进行CWS。

- **优势**：BERT的subword分词能够处理大部分保险术语，无需额外的分词器
- **优化建议**：
  - 如果保险领域有大量生僻术语，可考虑在预训练模型中加入领域词典
  - 或使用`jieba`等分词工具进行辅助，将保险术语作为自定义词典

### 2. 命名实体识别（NER）

**推荐方案**：复用现有`Model4s`模型，用于识别保险产品相关实体。

- **实体类型**：保险产品名称、保险公司、保障责任、保费、保障期限等
- **实现方式**：通过`Model4s`预测实体的start/end位置，实现实体识别
- **优化建议**：在训练数据中增加保险实体的多样性，提高模型泛化能力

### 3. 关系抽取（RE）

**推荐方案**：复用现有`Model4po`模型，用于识别保险实体之间的关系。

- **关系类型**：所属公司、产品类型、保费、保障期限、保障责任等
- **实现方式**：通过`Model4po`预测关系类型和对象的start/end位置，实现关系抽取
- **优化建议**：根据保险领域特点，调整关系类型数量和定义

## 四、实施步骤

### 1. 准备工作

1. **数据收集**：收集保险产品相关的文本数据
2. **数据标注**：按照`spo_list`格式标注训练数据
3. **定义关系类型**：创建`insurance_predicate.json`文件，定义保险领域的关系类型
4. **选择预训练模型**：下载并准备保险/金融领域的预训练BERT模型

### 2. 代码调整

1. **修改配置**：更新`config`类中的路径和参数
2. **调整数据加载**：根据需要调整`load_data`函数，适应保险数据格式
3. **优化分词处理**：如有需要，加入保险领域自定义词典

### 3. 模型训练

1. **初始化模型**：使用保险领域预训练模型初始化`Model4s`和`Model4po`
2. **模型微调**：使用保险标注数据训练模型
3. **模型评估**：使用测试数据评估模型性能，调整超参数

### 4. 模型部署与优化

1. **模型导出**：保存训练好的模型
2. **推理测试**：使用实际保险产品文本测试模型性能
3. **持续优化**：根据实际使用情况，不断优化模型和训练数据

## 五、性能评估指标

使用以下指标评估模型性能：

- **F1分数**：综合考虑精确率和召回率，是关系抽取任务的主要评估指标
- **精确率（Precision）**：模型预测正确的三元组占所有预测三元组的比例
- **召回率（Recall）**：模型预测正确的三元组占所有真实三元组的比例
