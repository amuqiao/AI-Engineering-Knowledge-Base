## 统一大模型网关（LLM Gateway）实现方案

### 核心架构设计

```
客户端请求
    ↓
[接入层] Nginx/API Gateway
    ↓
[LLM Gateway 核心层]
  ├── 鉴权模块
  ├── 限流模块

  ├── 计费模块
```

### 一、多模型鉴权（Auth）

**实现思路：**

```python
# 统一鉴权中间件
class AuthMiddleware:
    def __init__(self):
        self.api_key_store = RedisStore()  # 存储用户 API Key
    
    async def verify(self, request):
        api_key = request.headers.get("X-API-Key")
        
        # 1. 验证 key 是否有效
        key_info = await self.api_key_store.get(api_key)
        if not key_info:
            raise UnauthorizedException("Invalid API Key")
        
        # 2. 验证 key 是否有访问该模型的权限
        model = request.body.get("model")
        if model not in key_info["allowed_models"]:
            raise ForbiddenException("Model not allowed")
        
        # 3. 验证 key 是否过期
        if key_info["expires_at"] < now():
            raise UnauthorizedException("API Key expired")
        
        return key_info
```

**关键能力：**
- API Key 多级权限（用户级 / 团队级 / 企业级）
- 模型白名单控制
- Key 有效期管理 + 自动轮转
- JWT Token 支持（短期访问令牌）

### 二、流式限流（Streaming Rate Limit）

流式场景比普通 HTTP 复杂，需要同时限制 **请求频率** 和 **Token 消耗速率**：

```python
class StreamingRateLimiter:
    def __init__(self, redis_client):
        self.redis = redis_client
    
    # 令牌桶算法：限制 RPM（每分钟请求数）
    async def check_rpm(self, user_id: str, model: str) -> bool:
        key = f"rpm:{user_id}:{model}"
        current = await self.redis.incr(key)
        if current == 1:
            await self.redis.expire(key, 60)
        
        limit = await self.get_rpm_limit(user_id, model)
        return current <= limit
    
    # 滑动窗口：限制 TPM（每分钟 Token 数）
    async def check_tpm(self, user_id: str, tokens_used: int) -> bool:
        key = f"tpm:{user_id}"
        now_ts = time.time()
        
        # 使用 Redis Sorted Set 实现滑动窗口
        pipe = self.redis.pipeline()
        pipe.zremrangebyscore(key, 0, now_ts - 60)
        pipe.zadd(key, {f"{now_ts}:{tokens_used}": now_ts})
        pipe.zrangebyscore(key, now_ts - 60, now_ts)
        results = await pipe.execute()
        
        total_tokens = sum(int(r.split(":")[1]) for r in results[2])
        return total_tokens <= self.get_tpm_limit(user_id)
    
    # 流式响应中实时累计 token 消耗
    async def stream_with_limit(self, response_stream, user_id):
        total_tokens = 0
        async for chunk in response_stream:
            tokens_in_chunk = estimate_tokens(chunk)
            total_tokens += tokens_in_chunk
            
            # 超限则中断流
            if not await self.check_tpm(user_id, tokens_in_chunk):
                yield build_error_chunk("Rate limit exceeded")
                return
            
            yield chunk
```

### 三、模型路由与负载均衡

```python
class ModelRouter:
    def __init__(self):
        self.providers = {
            "gpt-4o": [OpenAIProvider(), AzureProvider()],
            "claude-3-5-sonnet": [AnthropicProvider()],
            "qwen-max": [AlibabaProvider()],
        }
    
    async def route(self, request):
        model = request.model
        providers = self.providers[model]
        
        # 策略1：轮询负载均衡
        provider = self.round_robin(providers)
        
        # 策略2：基于延迟的智能路由
        provider = await self.latency_based_routing(providers)
        
        # 策略3：基于成本的路由（低优先级请求走便宜模型）
        provider = self.cost_based_routing(request.priority, providers)
        
        return provider
```

### 四、计费监控（Billing & Metering）

```python
class BillingService:
    async def record_usage(self, event: UsageEvent):
        # 1. 实时写入计费记录
        await self.db.insert("usage_logs", {
            "user_id": event.user_id,
            "model": event.model,
            "prompt_tokens": event.prompt_tokens,
            "completion_tokens": event.completion_tokens,
            "cost": self.calculate_cost(event),
            "latency_ms": event.latency_ms,
            "timestamp": now()
        })
        
        # 2. 更新用户余额（预付费模式）
        cost = self.calculate_cost(event)
        new_balance = await self.redis.decrbyfloat(
            f"balance:{event.user_id}", cost
        )
        
        # 3. 余额预警
        if new_balance < self.get_warning_threshold(event.user_id):
            await self.notify_low_balance(event.user_id, new_balance)
        
        # 4. 余额耗尽熔断
        if new_balance <= 0:
            await self.suspend_user(event.user_id)
    
    def calculate_cost(self, event):
        price_table = {
            "gpt-4o": {"input": 0.005, "output": 0.015},  # per 1K tokens
            "claude-3-5-sonnet": {"input": 0.003, "output": 0.015},
        }
        prices = price_table[event.model]
        return (event.prompt_tokens * prices["input"] + 
                event.completion_tokens * prices["output"]) / 1000
```

### 五、降级策略（Fallback & Circuit Breaker）

```python
class FallbackStrategy:
    def __init__(self):
        # 熔断器：连续失败 5 次后熔断 30 秒
        self.circuit_breaker = CircuitBreaker(
            failure_threshold=5,
            recovery_timeout=30
        )
        
        # 降级链：主模型 → 备用模型 → 本地模型
        self.fallback_chains = {
            "gpt-4o": ["gpt-4o-mini", "claude-3-haiku", "qwen-turbo"],
            "claude-3-5-sonnet": ["claude-3-haiku", "gpt-4o-mini"],
        }
    
    async def call_with_fallback(self, request):
        models = [request.model] + self.fallback_chains.get(request.model, [])
        
        for model in models:
            if self.circuit_breaker.is_open(model):
                continue  # 跳过熔断中的模型
            
            try:
                response = await self.call_model(model, request)
                self.circuit_breaker.record_success(model)
                
                # 记录是否发生了降级
                if model != request.model:
                    await self.metrics.record_fallback(
                        original=request.model, actual=model
                    )
                return response
                
            except (TimeoutError, ProviderError) as e:
                self.circuit_breaker.record_failure(model)
                continue
        
        raise AllProvidersFailedException("All models unavailable")
```

### 六、可以扩展的更多功能

| 功能模块 | 说明 |
|---|---|
| **Prompt 缓存（Semantic Cache）** | 对语义相似的请求返回缓存结果，降低成本（Redis + 向量相似度） |
| **请求/响应日志审计** | 完整记录所有对话内容，用于合规审计和问题排查 |
| **内容安全过滤** | 请求前检测违规内容，响应后过滤敏感信息 |
| **Prompt 注入防护** | 检测并阻断 Prompt Injection 攻击 |
| **请求重试 & 超时控制** | 自动重试、超时熔断，保障稳定性 |
| **多租户隔离** | 企业级租户隔离，独立的配额、计费、日志 |
| **A/B 测试路由** | 按比例将流量分配给不同模型，评估效果 |
| **成本优化路由** | 根据任务复杂度自动选择性价比最优模型 |
| **响应流式转非流式** | 统一接口，屏蔽不同厂商的流式差异 |
| **API 格式统一适配** | 统一转为 OpenAI 格式，下游无需感知多厂商 |
| **Observability 可观测性** | 集成 OpenTelemetry，接入 Prometheus + Grafana |
| **WebSocket 长连接管理** | 支持长会话场景的连接保活与管理 |
| **Mock 模式** | 开发测试时返回 mock 数据，不消耗真实额度 |

### 七、推荐技术栈

```
接入层：     Nginx / Envoy
网关框架：   FastAPI (Python) / Gin (Go) — 推荐 Go 性能更好
缓存/限流：  Redis (令牌桶 / 滑动窗口)
消息队列：   Kafka (异步计费事件)
数据库：     PostgreSQL (用户/计费) + ClickHouse (日志分析)
监控：       Prometheus + Grafana + Loki
追踪：       Jaeger / OpenTelemetry
部署：       Kubernetes + Helm
```
