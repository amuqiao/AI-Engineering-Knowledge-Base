## 深度学习模型分析提示词

分析深度学习模型 {模型名称}，输出一份专业技术分析文档。

---

### 分析维度（五个阶段，按序展开，每个阶段独立成章）

**① 模型定位**
- 一句话说清楚：属于哪个研究方向、解决什么任务、面向什么场景
- 核心创新点是什么（相对于前代或同期工作的突破在哪里）

**② 结构认知**
覆盖以下三个层次：

- **架构拆解**：按「功能模块 → 子模块 → 关键算子」三层拆解，说明各层职责边界
  和模块间连接方式（串行 / 并行 / 跨层特征复用）

- **数据直觉**：以 1 条具体样例为主线，完整展示数据在各阶段的形态变化：
  原始输入（真实内容）→ 预处理后（tokenization / patch / 归一化等）→
  关键中间表示（说明「这一步在表达什么」，不只是维度）→
  原始输出 → 后处理结果

- **核心数据流**：数据从输入到输出的完整流转路径，标注每个关键节点的张量维度变化

**③ 机制深挖**（按实际模型家族展开，未涉及的跳过）

- **若是 Transformer / 注意力机制** → 分析注意力变体（MHA/MQA/GQA）、
  位置编码方案（RoPE/ALiBi/绝对位置）、归一化策略、KV Cache 设计
- **若是扩散模型** → 分析正向加噪过程、噪声调度策略（线性/余弦）、
  条件控制机制（CFG/ControlNet）、采样算法（DDPM/DDIM/DPM-Solver）
- **若是 CNN / 检测网络** → 分析感受野演进逻辑、特征金字塔结构、
  归纳偏置设计，以及检测头和后处理（NMS）
- **若是 MoE（混合专家）** → 分析路由机制（Top-K 门控）、
  专家负载均衡策略、稀疏激活的训练稳定性问题
- **若是多模态模型** → 分析模态对齐机制、跨模态融合策略、
  对比学习目标（CLIP式）或生成式对齐
- **若是 SSM / 线性注意力** → 分析状态空间方程、并行扫描实现、
  与 Attention 的近似关系
- **若是自回归生成模型** → 分析解码策略（贪心/束搜索/温度采样/Top-p），
  推理时的 KV Cache 机制，以及长文本外推能力

> 对于命中的每种机制，遵循「直觉 → 原理 → 公式」的递进顺序：
> 先用 1~2 句直觉语言说清楚「这个机制本质上在做什么」，
> 再深入计算原理，最后给出数学公式（使用 $...$ / $$...$$ 语法）。

**④ 训练与评估**
- **训练策略**：损失函数设计与选用原因、优化器与学习率调度、
  关键训练技巧（数据增强 / 课程学习 / 梯度裁剪 / 混合精度等）
- **评估指标**：主要指标的含义与选用原因；在核心 Benchmark 上与
  代表性基线的对比；关键消融实验中各组件的贡献量化
- **效率指标**：参数量 / FLOPs / 推理延迟 / 内存占用

**⑤ 推理与部署**
- 推理阶段与训练阶段的关键差异（BN 模式切换 / Dropout 关闭 / 图优化等）
- 输出后处理流程（NMS / beam search / 解码链路，视任务而定）
- 常见部署优化手段（量化 / 蒸馏 / ONNX 导出 / TensorRT 加速，如有）

**⑥ 关键设计决策**
选取 5~8 个影响深远的设计选择，每条聚焦「为什么这样设计」而非「是什么」，
结构为：
> 背景 → 可选方案对比 → 选择原因 → 代价与补偿

**⑦ FAQ**
覆盖以下四类，共 12 题以上：
- 基本原理（核心概念和工作机制）
- 设计决策（架构选择背后的权衡）
- 实现细节（工程实现的注意事项）
- 性能优化（推理加速、内存优化、训练提速等）

---

### 质量标准
- **直觉优先**：任何数学描述之前必须有直觉解释（1~2 句）；
  禁止直接抛出公式，无直觉铺垫的公式视为质量不达标
- **数学格式**：行内公式使用 `$...$`，独立公式块使用 `$$...$$`；
  变量须有含义说明，不接受无注释的裸公式
- **维度准确性**：张量维度标注须与模型实际实现一致，
  不确定处标注"约"或给出估算依据
- **内容底线**：数据直觉章节必须使用具体样例（真实文本/图像内容），
  不接受用"输入 token 序列"等抽象描述代替

### 格式规范

输出格式为 Markdown，充分使用 Mermaid 图辅助说明。

**必须包含的图类型**（每类至少一张）：
- 模型整体架构图（建立模块全局认知）
- 前向传播 / 张量流图（追踪数据维度变化）
- 核心模块内部结构图（深入关键组件）
- 训练流程图（理解优化过程）
- 数据处理流水线图（理解输入预处理）

**遇到以下情况时主动添加补充图**
（无需逐一覆盖，判断"一张图能否替代三段文字"）：
- 注意力权重分布 / 位置编码 / 门控结构等非线性机制
  → 展示信息流与权重分配
- 扩散过程 / 噪声调度曲线 → 展示时间步与噪声量的关系
- 多分支 / 跨层连接 / 特征复用路径 → 文字难以清晰表达时
- 推理阶段流程与训练流程有显著差异 → 对比展示两条路径
- 消融实验中各组件的贡献对比 → 帮助理解设计决策的必要性
- 部署优化链路（量化 / 蒸馏 / 导出步骤）有具体分支判断时
- 任何"用文字描述需要三段以上才能说清"的结构关系

> 注：各模型家族的机制专属图（如 Transformer 注意力图、扩散过程图）
> 由「③ 机制深挖」负责，此处补充图专注于**执行复杂度**和**结构可视化**场景。

**每张图的配套文字规范**：
- 图前导语（必须）：1~3 句话说明关注焦点和阅读路径
- 图后要点（复杂图必须）：2~4 条说明图中非显而易见的设计决策；
  节点少于 8 个的纯示意图可省略

**Mermaid 作图风格**参考 [mermaid深度学习模型分析规范与参考.md]

**架构拆解格式**：使用 ASCII 树形结构（`├──` / `└──` / `│`），
每行末尾附简短功能说明，禁止使用表格替代树形结构。

### 输出前自查清单

完成初稿后，对照以下问题检查，有遗漏则补充：

- [ ] 是否识别了模型所属的家族类型，并在「③ 机制深挖」中逐一展开？
- [ ] 数据直觉是否用了具体样例，而非抽象维度描述？
- [ ] 张量流图是否标注了每个节点的维度变化？
- [ ] 每个 FAQ 是否真正回答了「为什么」而非只描述「是什么」？
- [ ] 每张 Mermaid 图是否有图前导语和图后要点？
- [ ] 关键设计决策是否包含「可选方案对比」而非只陈述最终选择？
- [ ] 所有数学公式是否使用了正确的 $...$ / $$...$$ 语法，且有变量含义说明？
